Properties can be used in Bases for organising your notes.



| Propery          | Implementation                                                                                                                                                                       |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| *topics*:        | the key terms on the topic your note covers like: exercise, bible, scrum, etc.                                                                                                       |
| *category*:      | the umbrella area or group that the note belongs to. you can create specific templates for notes of particular categories. see `Templates/Category` for example category type notes. |
| *related*        | use this to add related sources like **URLs** or links to other note with `[[note_name]]`                                                                                            |
| **aliases**      | are unique to each note. They add an alternative name to that file.                                                                                                                  |
| *content_origin* | nice to have... to track if you created your own note or copied it or used ai to generate it or used a mix of ai and own                                                             |
